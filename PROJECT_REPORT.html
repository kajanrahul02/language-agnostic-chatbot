1. Abstract

Polyglot AI is an advanced, production-ready, full-stack language-agnostic AI chatbot. In modern software engineering, providing natural, high-performance interactions in regional languages remains a bottleneck. This project resolves that barrier by wrapping high-intelligence LLMs (Google Gemini SDK) with local Web Speech Synthesis/Recognition APIs and Firebase Cloud integration. Polyglot AI instantly detects user inputs in over 100 languages, translates input to English for consistent internal log processing, tracks contextual metrics, and streams fluent responses back in the user's native tongue.

2. Objectives & Scope

The main goals of Polyglot AI consist of:

  • Universal Language Support: Detect, translate, and synthesize speech patterns dynamically for a global user base.
  • Dynamic AI Routing Engine: Intelligent routing logic to choose between different model levels (Lite, General, Thinking, Maps Grounding) based on intent.
  • Cloud Synchronization: Live user session authentication and storage of history files over Firebase Cloud Services.
  • Premium Interactive Interface: Design a dark glassmorphic responsive interface utilizing negative space, custom metrics, and fluid feedback.

3. System Architecture & Flowchart

The application follows a full-stack decoupled architecture which secures the Google Gemini API key on the server side (preventing developer token compromises in client-side devtools) while using the client for intensive media, UI, and speech processing.

4. Technical Implementation & Features

4.1 Dynamic Model Orchestration (Google GenAI)

In server.ts, the API requests are dynamically evaluated to assign the optimal Gemini model, ensuring minimal cost overhead and maximal logic depth. The models are selected based on these specific routing configurations:

  • Vision Analysis: When an image is attached, the request is mapped to gemini-3.1-pro-preview for detailed optical parsing.
  • High Thinking Mode: For hard math, programming, or coding logic, we trigger gemini-3.1-pro-preview with ThinkingLevel.HIGH configurations.
  • Maps Grounding: For real-time locations and directions, the chatbot leverages gemini-3.5-flash with the active googleMaps tool integration.
  • Lite Mode: Super quick interactions utilize the lightweight gemini-3.1-flash-lite model.

4.2 Firebase Cloud Sync

The user's profiles, login credentials, and historical logs are persisted on Google Cloud Firestore and Authentication modules. The app secures direct read/write paths by establishing unique security boundaries for users (OIDC, Firebase, Email/Password, and Guest Sessions).

4.3 Web Speech STT & Accent TTS

Voice signals are parsed locally using browser microphones, converting them to text instantly. The AI's outputs are fed back to custom-built SpeechSynthesisUtterance routines that select native audio vocalizers matching the text's detected language code (such as Hindi, Spanish, or Japanese voices).

5. Database Schema & Architecture

Polyglot AI maintains standard, relational-like nested collections inside Firebase's NoSQL Firestore database to allow lightweight, real-time sync across clients.

// Session Collections Schema
/sessions/{sessionId} = { "title" : string, // Detected conversation title "userId" : string, // Firebase Auth user reference UID "createdAt" : timestamp, // ISO Session creation time "lastActiveAt" : timestamp // Activity touchstone }
// Message Collections Schema
/messages/{messageId} = { "sessionId" : string, // Session reference "userId" : string, // Owner identity UID "text" : string, // Chat message text "sender" : "user" | "bot", // Identity token "image" : string, // Base64 data if upload was present "mode" : string, // Model configurations used "timestamp" : number, // Epoc Unix time milliseconds "sentimentScore": number // Score compiled from 0 to 100 }
